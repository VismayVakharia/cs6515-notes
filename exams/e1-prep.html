6.5 Pebbling a Checker board
...

Sub-problem Definition
...

L[i, j] denote the max sum possible by placing pebbles on first i columns with i-th column having type_j (configuration-j) for 1 <= i <= n, 1 <= j <= 8
Define a function val(i, j) = sum of pebble covered integers by using type_j on column i

Recurrence Relation
...

L[1, j] = val(1, j) for 1 <= j <= 8
L[i, j] = val(i, j) + max_k{L[i-1, k]: k & j are compatible, 1 <= k <= 8} for 2 <= i <= n
return(max(L[n, .]))

Pseudo code
...

L = [1..i, 1..8] 2D array of size n x 8
for j = 1...8
	L[1, j] = val(1, j)
for i=2...n
	for j=1..8
		max_val = 0
		for k=1..8
			if compatible(k, j)
				max_val = max{max_val, L[i-1, k]}
		L[i, j] = val(i, j) + max_val
return(max(L[n, .]))

Run-time Analysis
...

O(n)

6.6 Weird Multiplication
...

Sub-problem Definition
...

L[i, j] be a set of symbols that can be a possible result of multiplying the substring s[i...j] for 1 <= i < j <= n
Let C(x, y) be the multiplication function: ab=b -> C(a, b) = b, for x,y in {a,b,c}

Recurrence Relation
...

L[i, i] = C(i, i) for 1 <= i <= n
L[i, j] = {C(x, y): x \in L[i, k], y \in L[k+1, j], i <= k < j} for 1 <= i < j <= n
return(a \in L[n, n])

Pseudo Code
...

L = [1..n, 1...n, 3] A 3D n x n x 3 matrix where 3rd dimension of size 3 represents true/false membership of symbols in the corresponding set defined above (L[i, j, 1] = true means 'a' belongs in the L[i, j] set)
Sym = [a, b, c]
for i=1->n
	for j=1->n
		for k=1->3
			L[i, j, k] = false
for i=1->n
	for k=1->3
		if Sym[k] = C(i, i)
			L[i, i, k] = true
for s=1->n-1
	for i=1->n-s
		j = i + s
		for k=i->j
			for x=1->3
				for y=1->3
					for k=1->3
						if Sym[k] = C(Sym[x], Sym[y])
							L[i, j, k] = true
return(L[n, n, 1])

Run-time Analysis
...

O(n^3)

...

Sub-problem Definition
...

L[i, j] = length of the longest common subseq in substring s[1...i] and s[j...n] (reversed s'[1...j]), for 1 <= i,j <= n

Recurrence Relation
...

L[0, j] = 0, L[i, n+1] = 0, for 0 <= i <= n, 1 <= j <= n+1
L[i, j] = { 1 + L[i-1, j+1] if s[i]=s[j]; for 1 <= i,j <= n
{ max{L[i-1, j], L[i, j+1]} otherwise; for 1 <= i,j <= n
return(L[n, 1])

Pseudo Code
...

L = [0...n, 1...n+1] a 2D matrix of size n+1 x n+1
for i=0->n
	L[i, n+1] = 0
for j=1->n+1
	L[0, j] = 0
for i=1->n
	for j=n->1
		if s[i] = s[j]
			L[i, j] = 1 + L[i-1, j+1]
		else
			L[i, j] = max{L[i-1, j], L[i, j+1]}
return(L[n, 1])

2.15 In-place Split
...

First go through the array once to find the index of the pivot value v. Then, swap the pivot with the last element of the array. Now go through the array again keeping track of the index of left most element that has higher value than pivot. If an element is smaller than or equal to the pivot, swap it with the maintained indexed element and update the index by moving it to right by one. If the element is greater than the pivot then continue with the iteration. Once the iteration through the entire array is done, swap the element on tracked index and the last index (pivot).

2.17
...

Algorithm
...

Since the input is sorted array of distinct integers, a modified binary search would be applicable. We compare mid element with its index, if A[mid]=mid then we found such an index and return mid, if A[mid]<mid then the value is smaller than the index, so we recurse on the right half of the array by setting low=mid+1 and if A[mid]>mid the we recurse on the left half of the array by setting high=mid-1. If binary search returns without finding an index, then it doesn't exist.

Correctness
...

The input is sorted array of distinct integers. Being a sorted array allows for using Binary Search. And, the array doesn't have repeated integers (distinct) so once the value of an element is higher than its index, it will have to stay that way for the rest of the array. Thus, the values and index can cross (A[index]=index) at most once (it can stay at A[index]=index for multiple indices but once it is not, it can't become true again) and on the left of the crossing A[index]<index, and A[index]>index on the right.. Using this property, we perform binary search with the target value being the index itself (A[mid]=mid). If they match, then we found such an index and is returned. If A[mid]<mid then there's still chance for values to catch up to the index by skipping integers, thus we recurse on the right half of the array and if A[mid]>mid the recursion is done on the left half of the array. We split the search space in half at each iteration and if such index exists, binary search will find it and otherwise binary search returns empty and thus such index doesn't exist.

Run-time
...

The algorithm is a modified version of binary search with changing the target, the condition itself if single if statement taking O(1) time and everything else is just the standard binary search. Thus the total run-time of this algorithm is O(log n).

2.19 k-way merge
...

  • In first strategy, we use merge procedure from merge sort on first two arrays, then merge in third and so on. The first merge procedure merges two sorted arrays of length n each, thus the time required would be 2n (scans). Then, sorted array of 2n length is merged with another sorted array of length n resulting into and sorted array of length 3n thus 3n scans. Continuing this, the total number of scans performed are: 2n + 3n + 4n + ... + kn -> O(nk^2)

Algorithm
...

The input is k sorted arrays of size n. Since, we are merging already sorted arrays, the merge procedure from merge sort can directly be utilized. We make k/2 pairs of arrays arbitrarily and merge them. The result would be k/2 sorted arrays of size 2n. Now, we again pair-up arrays into k/(2^2) pairs and merge them using the merge procedure to result into k/4 sorted arrays of size 4n. Keep doing this dividing (pairing up) and conquering (merging) until we're left with a single sorted array of size kn.

Correctness
...

The input is k sorted arrays each with n elements. The merge step from merge sort takes in two half sized sorted array and merges them into a single sorted array. This algorithm is exactly like the merge sort except that the partitioning to the base case is already done and the building up using the merge is done partially such that there are k sorted sub-arrays of size n. Continuing the merging as in the merge sort, we pair up all arrays into pairs of 2 and merge them. The result would be k/2 sorted arrays of length 2n. Since the input and output of the merge procedure match the pattern, doing the same process repeatedly will finally give us an single sorted array of size kn.

Run-time
...

The merge procedure from merge sort performs in linear time. At the first iteration, we merge k/2 pairs of size n -> k/2 (2n) = kn operations. At next level of iteration, k/4 pairs are merged of size 2n -> (k/4) (4n) = kn operations. Thus, each level of recursion costs kn operations and at each level the number of array-pairs is cut in half until we reach single array. Thus, the recursion depth for this algorithm is log k. And total run-time is O(nk log k)

2.22 k-th element in union of 2 sorted arrays
...

Algorithm
...

  • The input is 2 sorted arrays of size m and n respectively
  • If either of the arrays is empty (length zero), return the k-th element of the other array
  • Calculate the mid indices for both arrays, mid1 and mid2.
  • If k <= mid1 + mid2, then compare the values of respective mid, if A1[mid1] < A2[mid2] then ignore the right half of the second array and set high2 = mid2 and if A1[mid1] >= A2[mid2] then ignore the right half of the first array by setting high1 = mid1 and recurse on the remaining parts of the array.
  • If k > mid1 + mid2 then also compare the values, if A1[mid1] >= A2[mid] then ignore the first half of array2 by setting low2 = mid2+1 and if A1[mid1] < A2[mid2] then ignore the first part of array1 by setting low1 = mid1+1 and recurse on the remaining parts of the array with updated value of k, k_new = k - mid2 or k_new = k - mid1 depending on which array's half elements were discarded.

Correctness
...

The input is two sorted arrays of size m and n. The arrays being sorted allows for using a binary search based approach. In base case, if either of the array is empty then the k-th smallest element will be the k-th smallest element in the other array, thus A2[k] would be the answer.
In the first case of k <= mid1 + mid2 and A1[mid1] <= A2[mid2], there are mid2 elements smaller than or equal to A2[mid2] and since A1[mid1] <= A2[mid2] another mid1 elements are also smaller than or equal to A2[mid2], thus total number of elements that are smaller than or equal to A2[mid2] are mid1 + mid2 <= k and thus, k-th smallest element can not be beyond A2[mid2] and can safely be discarded (by setting high2 = mid2). By symmetry, the same argument can be made for the case of A2[mid2] < A1[mid1]
Conversely, if k > mid1 + mid2 and A1[mid1] <= A2[mid2] then, there are at least mid1 + mid2 elements larger than or equal to A1[mid1] thus it is confirmed that the elements to the left of A1[mid1] can not include k-th smallest element and thus can be discarded by setting low1 = mid1 + 1 but the recursion on the remaining arrays will have updated value of k_new = k - mid1 as the first mid1 elements are now ignored. By symmetry, the other case of A1[mid1] > A2[mid2] can be explained.
At each step of the recursion, the half of one of the arrays is discarded while ensuring that k-th smallest element remains within the remaining parts of the arrays, and when base case is reached, the value is returned.

Run-time
...

At every level of the recursion, one half of one of the arrays is discarded. Thus, the total recursion depth can be log n + log m and the computation performed on each level is just two if-statement evaluation (O(1)) this, the total runtime of this algorithm is O(log m + log n)
